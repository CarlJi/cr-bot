创建优雅的`git commit`，在大型协作项目中非常重要。

**commit-check** 重点推荐以下两点实践：
* 不要有重复的`commit message`
* 常用`git rebase`命令，消除多余的`Merge`记录，也会让`git history timeline` 更加的优雅，少分叉
